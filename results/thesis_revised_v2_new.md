# 第一章 绪论

## 1.1 研究背景

水稻是全球最重要的粮食作物之一。联合国粮食及农业组织（FAO）的公开资料指出，全球有超过 35 亿人口依赖稻米获取至少 20% 的每日热量摄入[1]。与此同时，植物病虫害持续威胁粮食安全。FAO 指出，植物病虫害每年可导致全球作物产量损失约 20%-40%[2]。因此，在水稻生产场景中建立高效、可扩展的病害检测系统具有明确的现实意义。

传统的水稻病害诊断主要依赖植物病理学专家的田间目视检查。这种方式通常劳动密集、耗时，并且高度依赖专家经验，难以满足大面积农田的快速巡检需求[3]。此外，专家资源在空间分布上并不均衡，偏远地区往往难以及时获得可靠诊断，进一步限制了传统方法的可扩展性。

近年来，深度学习在计算机视觉领域的进展为农业病害检测提供了新的技术路径。在标准视觉分类基准上，卷积神经网络（Convolutional Neural Network, CNN）已达到甚至超过经典的人类基线表现[4]。然而，将此类方法直接迁移到水稻病害检测任务时，仍然面临一个关键瓶颈：高质量监督标注的获取成本。

当前主流的检测或分割方法通常依赖边界框标注（bounding box annotation）或像素级标注（pixel-level annotation）。这类标注需要专家逐区域甚至逐像素标记病变位置，成本显著高于图像级标签。在公开视觉数据集中，密集标注往往需要每张图像数分钟乃至更长时间，标注成本会随数据规模迅速上升[5]。对于农业场景中的大规模数据构建而言，这种成本是弱监督方法得以成立的直接动机之一。

相比之下，图像级标签（image-level label）只需要回答"图中出现了哪一类病害"，获取门槛显著更低，也更接近真实农业场景中的数据收集方式。由此便引出本研究的核心问题：**是否能够仅利用图像级标签，训练出一个既能完成病害分类、又能输出具有空间意义的响应热力图与候选检测框的系统？**

弱监督学习（weakly supervised learning）正是针对这一问题的重要技术路线。其中，多示例学习（Multiple Instance Learning, MIL）尤其适合处理"只有整体标签、没有局部标注"的情形。本文将每张水稻图像视为一个 bag，将其切分后的多个局部区域视为 instances，在仅使用图像级标签的条件下学习发现与病害最相关的关键区域，从而实现从粗粒度类别标签到空间响应热力图及候选检测框的映射。

## 1.2 问题陈述

将弱监督 MIL 框架应用于水稻病害检测，至少面临以下几个关键挑战。

第一，**标签噪声（label noise）问题显著**。在 MIL 设定下，正袋中的多数切片并不一定真实包含病变区域，它们可能对应健康叶片、背景纹理或不相关结构。如果对所有切片施加同等强度的监督，模型会被大量伪正样本污染。

第二，**正样本内部质量差异大**。即使都来自正袋，不同切片对病害类别的表征强度也可能完全不同：有的切片包含典型且清晰的病灶，有的只包含模糊边缘或早期征兆。将这些切片等价看待，会削弱监督信号的有效性。

第三，**硬件资源受限**。本研究的训练与推理均在单张 NVIDIA RTX 4060 Laptop GPU（8GB VRAM）上完成[6]。这意味着模型设计必须兼顾可训练性、显存占用与推理成本。

第四，**病灶尺度变化显著**。水稻病害既可能表现为局部小斑点，也可能呈现为叶片或叶鞘上的大面积异常区域。因此，模型不仅需要识别病害类别，还需要具备处理多尺度空间模式的能力。

第五，**缺乏局部标注真值（local annotation ground truth）**。在仅有图像级标签的前提下，本文虽可输出空间热力图与候选检测框，但由于缺乏人工边界框或像素级掩码作为参照，无法直接采用标准目标检测中的 mAP 或语义分割中的 IoU / Dice 等指标对定位结果进行严格评估，只能结合 bag 级、tile 级、负样本相关指标与可视化结果进行间接验证。

## 1.3 研究目标

本研究的总体目标是：**设计并实现一个基于弱监督多示例学习的水稻病害分析框架，在仅使用图像级单标签的条件下，实现多病害分类，并生成与病害相关的空间响应热力图及候选检测框，为后续精准定位与决策提供弱监督线索。**

围绕这一总体目标，本文拟完成以下具体任务：

1. 提出一种非对称 MIL 训练框架（Scout-Snipe），将示例搜索与参数学习解耦，以降低标签噪声对梯度更新的污染。
2. 设计一种三级质量感知损失机制，并引入基于 bag 内部约束的软标签蒸馏策略，以提升弱监督下的学习稳定性。
3. 构建一套适用于单卡资源约束的多尺度推理流程，实现从原始图像到空间响应热力图与候选检测框的端到端输出。
4. 通过系统消融实验验证训练策略、骨干网络与混合组件的实际贡献，并分析框架性能的主要驱动因素。

## 1.4 研究范围与限制

### 1.4.1 **研究范围**

- 本研究针对 8 类常见水稻病虫害与健康背景共 9 个类别开展弱监督检测研究。具体类别包括：细菌性叶枯病（Bacterial Leaf Blight）、褐斑病（Brown Spot）、稻曲病（False Smut）、叶瘟（Leaf Blast）、穗颈瘟（Neck Blast）、稻叶甲（Rice Leaf Beetle）、稻纵卷叶螟（Rice Leaf Folder）、纹枯病（Sheath Blight），以及健康背景（Background）。
- 数据集包含 2,696 张正样本图像与 1,126 张负样本图像。经多尺度切片后，正样本池包含 150,311 个 tiles（训练集 97,527 / 验证集 52,784），负样本池包含 46,382 个 confirmed-clean negative tiles（训练集 37,105 / 验证集 9,277）[6]。
- 所有训练与推理均在单张 RTX 4060 Laptop GPU（8GB VRAM）环境下完成[6]。

**研究限制**

- 由于缺乏像素级真值，本文无法直接报告标准检测任务中的 mAP 或像素级 IoU 等指标，而需通过间接指标与可视化结果评估空间定位质量。
- 当前建模假设默认每张正样本图像只包含一种主导病害，尚未系统处理多病害共存情形。
- Tile 级分类模型在训练集外的数据上表现良好（包括最弱类别 BLB 在各尺度下 top-K tiles 均能以接近 100% 置信度正确分类），但多尺度融合推理流水线的检测框聚合阶段仍存在优化空间：不同类别的热力图响应在空间维度叠加后，阈值化与 NMS 过程可能放大次要类别的弱响应，导致最终检测框中混入非主导类别的候选。这是推理后处理层面的问题，而非分类能力本身的不足。
- 部分形态学相似的类别（如 BLB 与 Leaf Blast）在热力图层面存在交叉激活现象——即使 tile 级分类正确率极高，目标类别的热力图通道仍可能在非目标区域产生弱响应，这构成了当前检测框精度的主要瓶颈之一。未来引入具有记忆与状态感知能力的自适应推理模型有望缓解此问题。
- 本文输出形式为基于弱监督学习得到的空间响应热力图与候选检测框，研究目标不包括像素级病灶边界恢复，因此不将其定义为语义分割或实例分割任务。

## 1.5 论文贡献

结合现有材料，本文的主要贡献可概括为：

1. 提出 **Scout-Snipe 非对称 MIL 框架**，将"搜索关键实例"与"利用关键实例更新参数"分离，以缓解弱监督条件下的标签噪声污染。
2. 提出 **三级质量感知损失机制** 与 **Per-Bag KL 蒸馏策略**，使监督信号能够随样本质量动态调整，并避免跨 bag 混合蒸馏造成的热力图弥散。
3. 在单卡资源约束下实现了 **标准 CNN 主干 + Conv1×1 空间分类器 + 全局最大池化** 的轻量化主线架构，并通过消融结果显示：相较于引入 FPN、ViT 等混合组件，训练策略本身更可能是性能提升的主要来源[6]。
4. 设计了从多尺度切片、尺度加权融合、背景感知重权重到检测框提取的完整推理流水线，使仅有图像级标签的弱监督模型能够输出原图级空间热力图与候选检测框，从而为病灶区域提供可解释的定位线索[6]。
5. 对若干失败尝试（如 Negative Rejection 导致的 Silence Spiral、Feature Critic在预训练特征空间的失效、混合架构训练复杂度等）进行了系统记录，为后续研究提供可复用的经验边界[6]。

本章系统回顾与本文研究密切相关的已有工作，为后续方法设计提供理论基础与方法论依据。首先回顾水稻病害检测从传统图像处理到深度学习方法的演进历程（2.2 节）；其次梳理多示例学习（Multiple Instance Learning, MIL）的经典定义、方法发展脉络及其在医学与农业领域的应用现状（2.3 节）；随后介绍弱监督目标定位的核心技术——类激活映射（Class Activation Mapping, CAM）及其变体，以及弱监督语义分割的代表性框架（2.4 节）；接着讨论本文涉及的骨干网络架构与多尺度特征融合方法（2.5 节）；然后介绍知识蒸馏与软标签学习的基本原理（2.6 节）；最后在综述基础上总结现有研究空白，并定位本文工作的具体切入点（2.7 节）。

## 2.2 水稻病害检测的传统方法与深度学习方法

水稻病害检测的早期研究主要依赖手工设计的特征描述子与传统分类器。典型流程包括：基于颜色空间变换（如 HSV、L*a*b*）的病斑区域分割，基于灰度共生矩阵（Gray-Level Co-occurrence Matrix, GLCM）或局部二值模式（Local Binary Pattern, LBP）的纹理特征提取，以及基于 SVM、KNN 等浅层分类器的病害类别判定。此类方法在控制条件较强的场景下（如实验室采集、均匀光照、单一背景）可以取得较好效果，但对光照变化、复杂田间背景、病斑形态多样性及成像设备差异的鲁棒性不足[3]。Ismail 等人的综述指出，传统方法通常需要较强的领域知识支撑特征工程，且特征设计的可迁移性有限，制约了其在大规模实时检测场景中的可扩展性[3]。

随着深度学习在计算机视觉领域的广泛应用，基于卷积神经网络（Convolutional Neural Network, CNN）的端到端特征学习逐步替代了手工特征方法，成为水稻病害识别的主流技术路线。CNN 通过层级化的卷积—池化结构自动学习从低层纹理到高层语义的判别性表征，在高类内变异与复杂背景条件下通常优于传统方法[3]。在农业病害识别场景中，以 ResNet、VGG、DenseNet、EfficientNet 等为代表的骨干网络被广泛采用，并常结合 ImageNet 预训练权重的迁移学习策略，以缓解农业数据集规模有限的问题。

从方法论层面看，这一技术演进反映了特征获取范式的根本变化：传统方法依赖研究者显式设计特征提取规则，而深度学习方法通过数据驱动的方式自动发现与任务相关的判别模式。本文的工作沿着这一方向进一步推进，即在仅提供图像级标签的条件下继续降低监督成本，同时尽可能保留空间定位能力。

### 2.2.1 水稻病害深度学习检测方法的代表性工作

为更清晰地呈现当前技术格局，本节按监督粒度从强到弱的顺序，梳理水稻病害深度学习检测的三类主要方法。

**第一类：全监督图像级分类方法。** 这是目前文献中最常见的研究路线。研究者通常采用 ImageNet 预训练的 CNN 骨干网络，通过迁移学习在水稻病害数据集上进行微调，以实现病害类别的自动判定。Sethy 等人的综述系统梳理了 2007–2018 年间从传统颜色/纹理特征到 CNN 端到端方法的演进脉络[NEW-01]。近年来，注意力机制（如 CBAM、SE 模块）和轻量化架构（如 MobileNet、ShuffleNet）也被引入稻病分类任务[42][45]。在若干公开基准上，基于 EfficientNet 和 DenseNet 的迁移学习模型通常能够报告较高的图像级分类精度[41][43]。然而，图像级分类方法只能输出全局类别判定，无法回答“病害位于图像何处”这一空间问题，因此难以直接满足精准施药、局部巡检或病灶可视化分析等应用需求。

**第二类：全监督检测与分割方法。** 为获取空间定位信息，部分研究进一步采用目标检测框架（如 YOLO、Faster R-CNN）或语义分割架构（如 U-Net、DeepLab）。Patarapuwadol 等人在 NSTDA 数据集上提供了部分人工标注的局部病斑掩码作为定位评估基准[30]。然而，此类方法依赖边界框或像素级标注，标注成本显著高于图像级标签；在包含复杂背景的真实农田图像中，逐区域标注一张图像通常需要数分钟至数十分钟[5]，这会显著限制大规模数据集的构建效率与方法的实际可推广性。

**第三类：弱监督方法。** 弱监督学习旨在利用低成本标签（如图像级标签）逼近全监督方法的空间定位能力。在植物病害领域，弱监督研究已开始从图像级识别向局部定位与分割扩展。例如，Chen 等人（2023）将弱监督学习应用于马铃薯叶病识别[NEW-02]；Zhou 等人（2023）探索了弱监督框架在病害特征分割中的应用[NEW-03]。但在水稻病害场景中，将 MIL 框架与弱监督定位系统性结合的端到端方案仍较为有限。

## 2.3 多示例学习（MIL）

### 2.3.1 经典 MIL 范式与方法发展脉络

多示例学习（Multiple Instance Learning, MIL）是弱监督学习中的经典建模范式，尤其适合处理“整体标签可得、局部标注缺失”的监督结构。

Dietterich、Lathrop 与 Lozano-Pérez 在研究药物活性预测问题时首次形式化地提出了多示例问题（multiple instance problem）：每个训练样本由多个可能的构象（instance）表示，而只有其中部分构象真正决定该分子的活性标签[7]。随后，Maron 与 Lozano-Pérez 将这一问题抽象为一般性的 MIL 框架，并给出其核心假设：若一个 bag 至少包含一个正实例，则该 bag 标记为正；当且仅当 bag 中所有实例均为负时，该 bag 才标记为负[8]。这一“正 bag 至少一正、负 bag 全负”的非对称标签假设构成了经典 MIL 的理论基础。

需要说明的是，经典 MIL 假设最初主要针对二分类问题提出[7][8]。在多类别视觉任务中，研究者通常通过一对多（one-vs-rest）bag 标签建模、类别特定评分函数，或共享特征提取器配合多类别分类头等方式对其进行扩展。本文面向的多病害识别任务即属于这一扩展语境：bag 级监督来自图像类别标签，而实例级病灶位置仍保持未观测状态。

对于本文所面对的弱监督病害检测任务，MIL 的建模思想具有天然的适配性：一张带有图像级类别标签的水稻图像可被视为一个 bag，对其进行多尺度切片所得的局部区域则构成 bag 内的实例集合。MIL 框架为“从图像级标签出发发现关键病灶区域”提供了形式化的理论接口。

从 1997 年 Dietterich 等人的开创性工作至今，MIL 的方法论发展大致经历了三个阶段。

**第一阶段（1997–2010）：经典统计学习与大间隔方法。** 这一阶段的核心问题是在 bag 标签已知而实例标签缺失的条件下实现有效分类。代表性方法包括 Diverse Density（在实例特征空间中搜索最能同时解释所有正 bag、排除所有负 bag 的概念点）[8]、mi-SVM / MI-SVM[9]，以及基于 EM （Expectation-Maximization, 期望最大化）算法的迭代推断方法。需要注意的是，`mi-SVM` 与 `MI-SVM` 并非仅是记号大小写差异，而是 Andrews 等人原始文献中明确区分的两种 MIL-SVM 建模形式[9]。`mi-SVM` 采用实例中心（instance-centered）的最大间隔建模，将正 bag 中实例标签视为隐变量，在实例标签分配与 SVM 分类超平面的更新之间交替优化，更适合需要实例级判别的场景；`MI-SVM` 则采用 bag 中心（bag-centered）的最大间隔建模，通过 bag 内实例分数的最大值定义 bag margin，使正 bag 的标签主要由一个关键实例（witness instance）决定，更适合 bag 级分类任务[9]。对于本文关注的病害定位任务，如果目标包含潜在病灶区域的实例级识别，则 `mi-SVM` 的建模思想具有更直接的启发意义。

**第二阶段（2010–2018）：深度特征与注意力聚合。** 随着深度学习的兴起，MIL 的特征提取从手工设计转向端到端学习，聚合策略也从硬实例选择转向可微的软聚合。Ilse、Tomczak 与 Welling 于 2018 年提出的 Attention-based Deep MIL 是这一阶段的代表性工作[10]。该方法引入参数化注意力模块为每个实例计算标量权重，将 bag 表示建模为实例嵌入的加权和，使 bag 级预测对各实例的依赖程度可以被学习并参与梯度优化。同时期，MIL 与卷积网络的深度结合也成为重要趋势：研究者将 CNN 提取的 patch 级特征直接送入 MIL 聚合器，使视觉 MIL 能够处理高分辨率图像中的大规模实例集合。

**第三阶段（2018 至今）：大规模应用与训练策略创新。** 当前阶段的特点是 MIL 在数字病理（whole slide image analysis）、遥感弱监督目标检测及农业病害分析等领域得到更广泛应用[11]。方法创新主要集中在三个方面：(1) bag 内大量噪声实例的处理策略，如难例挖掘、课程学习与自步学习；(2) 更精细的聚合函数设计，如 Transformer-based aggregation、稀疏注意力机制；(3) MIL 与知识蒸馏、半监督学习等技术的结合，以提升弱监督下的标签效率。

本文提出的 Scout-Snipe 框架位于第三阶段的方法论语境中，属于训练策略创新这一方向——其并不改变聚合函数的基本形式，而是重新设计实例搜索与参数学习的时序组织关系，通过显式解耦两个过程来抑制噪声实例对梯度更新的污染。

### 2.3.2 基于注意力的 MIL

MIL 从传统的硬实例选择模型逐步演化为端到端可微分的深度学习框架，其中注意力聚合机制的引入是关键转折点。Ilse、Tomczak 与 Welling 提出的 Attention-based Deep MIL 将 bag 的标签概率建模为实例嵌入向量的加权聚合结果[10]。具体而言，该方法为每个实例 $h_k$ 计算一个标量注意力权重 $a_k$，并以 $z = \sum_k a_k h_k$ 构造 bag 表示，再将该表示输入 bag 级分类器。注意力权重由参数化网络生成，并经 softmax 归一化，从而在 bag 内形成概率分布[10]。

与传统的硬实例选择（如 max-pooling 或 top-K selection）相比，注意力聚合具有两方面优势。其一，它以可微分的方式保留 bag 内所有实例的信息，仅在聚合阶段通过权重分配体现差异化贡献；其二，注意力权重本身具有一定可解释性，可作为各实例对 bag 预测贡献度的近似指示。

基于注意力的 MIL 已成为数字病理、医学影像与遥感等多个弱监督场景中的主流聚合方案。本文的主线方法虽未采用标准的 attention pooling，但其目标与之相近：在弱标签条件下提高关键实例对 bag 预测的主导作用，同时抑制噪声实例的干扰。

### 2.3.3 MIL 在医学图像中的应用

MIL 在医学图像分析领域，特别是数字病理学（digital pathology）中的广泛应用，为农业弱监督定位任务提供了重要的方法论参照。Gadermayr 与 Tschuchnig 在其综述中指出，MIL 尤其适合处理全切片图像（whole slide image, WSI）这类仅具备 slide 级标签、而缺乏 patch 或像素级标注的高分辨率数据[11]。

例如，Campanella 等人构建了基于 MIL 的弱监督数字病理系统，仅使用病理报告诊断作为训练标签，在前列腺癌、基底细胞癌及乳腺癌淋巴结转移检测任务上取得较高判别性能，表明 slide 级标签可支持高性能 WSI 分类[NEW-05]。随后，Lu 等人提出 clustering-constrained-attention multiple-instance learning（CLAM）框架，在仅有 slide 级标签的条件下结合注意力聚合与实例级聚类约束，不仅提升了分类性能，也能够定位具有诊断价值的关键子区域[NEW-06]。

医学场景与农业场景在监督粒度结构上具有明显相似性：两者均面临“整体标签可得、局部精细标注昂贵或稀缺”的现实约束。数字病理领域已有研究表明，在 bag 级标签充分可靠的前提下，MIL 模型能够学习到具有空间意义的局部响应模式，即便缺乏像素级真值亦然[11]。这一结论为本文在仅有图像级标签的条件下追求空间定位能力提供了方法论上的先验支持。

### 2.3.4 MIL 在农业领域的初步探索

尽管 MIL 在医学病理领域已趋成熟，但将其迁移至农业病害检测并不能简单等同。两者的主要差异体现在三个方面：(1) WSI 的分辨率通常远高于农田图像（数万 × 数万像素 vs 数千 × 数千像素），但 WSI 的背景相对均匀，而农田图像的背景构成高度复杂，包括土壤、杂草、水面反光与阴影等；(2) 医学病理的典型任务多为二分类，而农业病害识别往往涉及多类别细粒度区分；(3) 医学领域已积累了较为标准化的公共 WSI 数据集（如 Camelyon16/17），而农业弱监督检测场景的标准化基准仍相对稀缺。

上述差异意味着，将 MIL 应用于水稻病害检测需要针对性的适配设计。本文的 Scout-Snipe 框架正是针对农业场景中的多类别、多尺度与高背景噪声等特点，在训练策略层面提出的系统性解决方案。

## 2.4 弱监督目标检测与定位

### 2.4.1 类激活映射（CAM）及其变体

在仅有图像级标签的条件下，如何从分类模型中恢复局部空间响应，是弱监督定位的核心技术问题。Zhou 等人提出的类激活映射（Class Activation Mapping, CAM）首次证明：当分类网络在最后一个卷积层之后采用全局平均池化（Global Average Pooling, GAP）并直接连接线性分类器时，可以通过分类器权重对最后卷积层的特征图进行线性加权，生成目标类别的空间响应图[12]。

然而，CAM 对网络架构施加了严格约束——要求最后卷积层与分类输出之间仅有 GAP 和线性层，这限制了其在任意网络结构上的适用性。Selvaraju 等人提出的 Grad-CAM 通过基于梯度的通道重要性估计策略解除了这一约束[13]。其核心思想是：对于目标类别 $c$，计算其分类得分 $y^c$ 相对于最后卷积层第 $k$ 个特征图 $A^k$ 的梯度，并对梯度在空间维度上做全局平均，得到通道权重 $$\alpha_k^c = \frac{1}{Z} \sum_i \sum_j \frac{\partial y^c}{\partial A^k_{ij}}$$ ；随后将特征图以该权重线性组合并施加 ReLU 激活，得到 $$L_{\text{Grad-CAM}}^c = \text{ReLU}\left(\sum_k \alpha_k^c A^k\right)$$ [13]。其中，ReLU 的作用是保留对目标类别具有正向贡献的空间区域，抑制可能属于其他类别的负向激活。Grad-CAM++ 进一步将通道权重的计算推广为像素级的高阶梯度加权，以改善多目标共存情形下的定位完整性[14]。

CAM 类方法在水稻病害检测中的价值在于，它们提供了一种从分类模型中恢复空间判别区域的机制，可用于验证模型是否捕获了具有病理学意义的特征。但这类方法也存在被广泛报告的结构性局限：由于分类目标仅要求识别“最具判别力”的局部区域即可获得正确标签，因此 CAM 热力图往往倾向于覆盖目标的判别性子区域，而非其完整空间范围。

### 2.4.2 弱监督语义分割（SEC 框架）

弱监督语义分割（Weakly-Supervised Semantic Segmentation, WSSS）旨在仅使用图像级标签等弱监督信号恢复像素级分割结果。Kolesnikov 与 Lampert 提出的 SEC（Seed, Expand and Constrain）框架是这一方向的代表性工作[15]。SEC 由三个协同作用的损失组件构成：

- **Seed（种子定位）损失**：以 CAM 生成的高置信区域作为初始前景线索，约束分割网络输出与 CAM 种子区域保持一致[15]。
- **Expand（区域扩展）损失**：引入全局加权秩池化（Global Weighted Rank Pooling, GWRP），通过对特征图排序响应进行加权聚合，推动分割网络突破 CAM 的局部最优覆盖，并扩展至更完整的目标区域[15]。
- **Constrain（边界约束）损失**：引入全连接条件随机场（Dense CRF）作为后处理先验，利用图像的低层外观一致性约束分割边缘，使其更贴合目标的真实视觉边界[15]。

SEC 的方法论原理已被后续多项农业病害弱监督分割研究借鉴[25]。Chen 等人提出的 C-CAM 进一步指出，标准 CAM 往往只能突出目标的判别性子区域，而难以恢复完整目标区域[16]。

弱监督分割与本文关注的弱监督病害定位共享一个基本矛盾：**分类任务仅需要发现“最能区分类别的局部区域”，而定位或分割任务则要求恢复“尽可能完整的目标空间范围”**。本文通过多尺度切片与 Per-Bag KL 蒸馏的组合策略，尝试在 MIL 框架内部缓解这一矛盾。

## 2.5 骨干网络与多尺度特征融合

本节选取 EfficientNet、FPN 与 Vision Transformer（ViT）作为综述对象，并非为了穷尽视觉模型的全部路线，而是因为它们分别对应本文实验体系中的三个关键角色：骨干特征提取、多尺度特征融合，以及全局上下文建模。

### 2.5.1 EfficientNet 系列

EfficientNet 系列由 Tan 与 Le 提出，其核心贡献在于提出复合缩放（compound scaling）策略——通过一个统一的缩放系数同时平衡网络深度、宽度与输入分辨率，从而在给定计算预算下系统性地搜索三者的较优配比[17]。这一设计使 EfficientNet 在 ImageNet 分类基准上实现了优于同参数量级模型的精度—计算量权衡。

EfficientNetV2 在此基础上进一步优化了训练效率[18]。其关键设计变更是在网络浅层引入 Fused-MBConv 模块，以替代标准的 MBConv（Mobile Inverted Bottleneck Convolution）。Fused-MBConv 将深度可分离卷积重新融合为标准卷积，虽然会增加理论 FLOPs，但在现代 GPU/TPU 等硬件上通常能够更充分地利用并行计算资源，从而提升实际训练速度[18]。这一架构变更在特征表征层面的意义将在第三章 3.4.1 节中结合本文实验进一步讨论。

本文主线实验采用 EfficientNetV2-S（22.15M 参数）与 EfficientNet-B0（5.29M 参数）两种骨干进行对比研究。

### 2.5.2 特征金字塔网络（FPN）

特征金字塔网络（Feature Pyramid Network, FPN）由 Lin 等人提出，是目标检测领域处理尺度变化问题的经典多尺度特征融合结构[19]。FPN 利用卷积骨干网络天然形成的多层级特征图——从浅层的高分辨率、低语义特征到深层的低分辨率、高语义特征——通过自顶向下路径（top-down pathway）与横向连接（lateral connections）将高层语义信息逐步传播到较高分辨率层，使各尺度特征同时具备更丰富的语义信息与空间细节[19]。

对于水稻病害图像，FPN 的多尺度融合思想具有现实意义：病灶尺度差异显著，从叶瘟的局部小斑点到纹枯病的大面积叶鞘异常均可能出现。本文实验性混合架构将 FPN 作为 neck 组件引入，其实际增益将在第四章的消融实验中讨论。

### 2.5.3 Vision Transformer（ViT）

Vision Transformer（ViT）由 Dosovitskiy 等人提出。该方法将输入图像切分为固定大小的非重叠 patch，并将每个 patch 线性映射为 token 嵌入后送入标准 Transformer 编码器[20]。ViT 的重要意义在于，它证明了纯自注意力架构在大规模预训练条件下可以达到甚至超过卷积网络的图像分类性能。然而，Dosovitskiy 等人同时指出，ViT 在中小规模数据集上的表现通常不及同参数量级的卷积网络，因为 Transformer 缺乏卷积网络固有的局部性（locality）与平移等变性（translation equivariance）等归纳偏置，需要更大规模数据来补偿[20]。

本文实验性混合架构将一个轻量 ViT 残差块放置在 FPN 降维后的 256 通道特征图上，以探索全局自注意力对弱监督定位任务的潜在贡献。该组件的消融实验结果将在第四章呈现。

## 2.6 知识蒸馏与软标签学习

知识蒸馏（Knowledge Distillation, KD）是一种利用教师模型（teacher）输出的软概率分布来监督学生模型（student）训练的技术。Hinton、Vinyals 与 Dean 在其开创性工作中指出，教师模型的软标签（soft targets）编码了类别之间的相对相似性结构——例如，某些错误类别的概率虽然较低但持续非零，反映了这些类别与目标类别在特征空间中的近邻关系[21]。这一“暗知识”（dark knowledge）无法从 one-hot 硬标签中直接获得。实践中，经典知识蒸馏常通过温度参数对 logits 进行平滑，并以 KL 散度（Kullback–Leibler divergence）衡量教师分布与学生分布之间的差异[21][22]。

在弱监督学习语境下，软标签机制还具有额外价值：当训练标签本身可能受噪声污染时，软概率分布相比 one-hot 硬标签能够提供更平滑、也更具不确定性感知能力的监督信号，有助于缓解模型对噪声标签的过拟合。本文提出的 Per-Bag KL 蒸馏策略借鉴了知识蒸馏的基本原理，但在两个关键层面进行了任务特化的适配：第一，教师信号不来自外部模型，而是来自同一 bag 内由 Scout 阶段识别出的最高质量 Top-1 实例预测分布；第二，蒸馏过程通过双门控机制严格限制在 bag 内部，避免跨 bag、跨类别的软标签混合导致热力图弥散。

## 2.7 文献综述总结与研究空白

综合以上各节的文献回顾，现有研究已在以下方向积累了较为成熟的方法基础：水稻病害识别从传统手工特征方法转向 CNN 端到端学习；MIL 为“整体标签 → 局部发现”的弱监督建模提供了系统性范式；CAM / Grad-CAM 及 WSSS 方法表明，仅使用图像级标签也可以恢复具有空间意义的类响应信息；EfficientNet、FPN 与 ViT 提供了不同设计哲学下的视觉骨干与多尺度建模方案；知识蒸馏则为概率分布层面的软监督迁移提供了理论工具。

然而，将上述技术直接组合并不能自动形成有效的端到端系统。基于本章综述，可以归纳出以下研究空白：

1. **水稻病害场景中的 MIL 弱监督定位研究仍不充分。** 现有农业文献仍以图像级分类或全监督检测为主，将 MIL 框架系统性用于水稻病害弱监督定位的工作相对有限。
2. **实例选择与参数学习的耦合仍是弱监督 MIL 训练中的脆弱环节。** 标准 MIL 流程通常同步执行“发现关键实例”与“利用该实例更新模型参数”两个过程，在训练早期容易因低质量实例选择而产生梯度污染。
3. **bag 内部实例质量差异的差异化监督缺少统一机制。** 现有方法主要通过注意力权重或伪标签质量过滤间接处理实例质量差异，尚缺乏显式的分层质量门控与 bag 内约束蒸馏策略。
4. **多尺度融合与软标签迁移在资源受限弱监督场景中的协同效果，仍缺乏任务驱动的系统性验证。**

### 3.2.3 多尺度切片策略（Scale Diversity）

本项目在离线切片与在线训练两个阶段均采用多尺度策略，确保模型对不同尺度的病灶具备鲁棒性。离线切片阶段使用三个尺度：768px、1024px、1536px，所有切片统一 resize 至 384×384 后存入 LMDB。推理阶段根据原图大小自适应选择 tile 尺寸集合：较大图像采用 [768, 1024, 1536]，较小图像采用 [384, 512, 768]。

Scale Diversity 机制在 Scout 阶段的 Top-K 选择中引入同尺度惩罚因子（penalty = 0.5）：当某一尺度的 tile 已被选入 Top-K 后，来自同一尺度的后续候选 tile 的得分将乘以 0.5 的折扣系数，从而鼓励 Top-K 集合包含来自不同尺度的 tiles，使模型在每个训练步中同时接触粗粒度语义和细粒度纹理信息。

### 3.2.4 负样本池构建

负样本池（negative pool）是整个训练框架的关键前提。项目的设计原则明确指出：**整个训练框架依赖于一个经确认无病害的干净负样本池**，否则模型将从自身尚不稳定的预测中自举，导致背景基线漂移[6]。负样本池规模为 46,382 个 tiles，按 768:1024:1536 的尺度分布为 26,213:14,443:5,726。

在训练中，负样本池承担两项作用：（1）为模型提供稳定的背景类别监督信号；（2）在 Scout-Snipe 训练中与正袋 Top-K 实例共同组成训练批次。此外，负样本池支持硬负样本挖掘（Hard Negative Mining）：每批采样 8 个负样本 tiles，其中 25% 来自硬负样本池（recall-failed tiles），75% 随机采样。硬负样本池还配备了生命衰减机制（life decay）和黑名单机制（blacklist），防止标注噪声或极端混淆样本长期污染训练。

### 3.2.5 数据增强

本研究采用基于 Albumentations 库的数据增强流水线，包括几何变换（随机水平/垂直翻转、随机旋转、随机缩放）和光度畸变（亮度、对比度、饱和度调整，色调偏移）。未引入 CutMix、Mixup 等混合增强策略，以避免在弱监督条件下引入额外的标签歧义。

值得强调的是，Scout-Snipe 框架对数据增强的一致性有严格要求：Scout 阶段选出的 Top-K tiles 在缓存时保留增强后的像素数据（通过 AugmentedTileCache），Snipe 阶段直接从缓存中取用，确保 Scout 与 Snipe 看到完全相同的增强结果。若两个阶段分别执行独立的随机增强，将导致 Top-K 选择与实际训练数据之间的不一致，破坏训练的数学完整性。

## 3.3 Scout-Snipe 非对称 MIL 训练框架

### 3.3.1 问题形式化

本文将每张输入图像表示为一个 bag，记作 $B = \{x_1, x_2, \ldots, x_N\}$，其中每个 $x_i$ 为一个 tile 实例。每个 bag 仅具备图像级类别标签 $y$，不提供实例级病灶位置标签。目标是学习一个实例级分类器，使其能够：（1）预测每个 tile 的类别归属；（2）在 bag 内部发现与该类别最相关的关键实例（Top-K 选择）；（3）将实例级分数投影为空间热力图，进而提取检测框。

与经典 MIL 框架不同，本文不采用 bag-level 聚合层进行端到端训练，而是直接在实例级别施加监督。这一设计决策将训练范式从标准的多示例学习（MIL）推进至弱监督目标检测（WSOD）：牺牲了 bag 层面的自动去重能力，但换取了推理端对空间分布信息的完全保留——实例分数可以自由重组为任意分辨率的病害分布图。

### 3.3.2 Scout 阶段：无梯度实例搜索

在 Scout 阶段，模型切换至 `eval()` 模式，对一个 bag 内的所有 tiles 执行无梯度前向推理（`torch.no_grad()`），为每个 tile 产生类别分数，随后选择 Top-K 个得分最高的 disease tiles 作为后续训练的候选实例（K=3）。选出的 tiles 连同其增强后的像素数据一并缓存至 AugmentedTileCache。

该阶段不产生反向传播，因此大量低质量实例的噪声预测不会直接污染模型梯度。`eval()` 模式确保 BatchNorm 层使用全局运行统计量（running statistics），而非当前批次统计量，从而保证 Top-K 选择的稳定性。

### 3.3.3 Snipe 阶段：有梯度精准训练

在 Snipe 阶段，训练真正发生在已被 Scout 选出的 Top-K 正样本 tiles 与来自负样本池的 negative tiles 上。模型切换至 `train()` 模式以启用梯度计算，但 BatchNorm 层保持 `eval()` 状态（Anti-Collapse 机制，详见 3.5.5 节）。

这一 BN 冻结策略的必要性源于 Scout-Snipe 的批次构成特性：Snipe 的训练批次是由来自不同图像的高激活 patches 拼接而成的非自然批次（non-i.i.d. batch），其统计分布与自然图像的批次统计量存在显著差异。BatchNorm 的核心假设是 mini-batch 的统计量能够近似总体分布[new-7]，而 Snipe 的选择性采样批次显然违反了这一假设。Wu 和 He 在 Group Normalization 的工作中已明确指出，当批次统计量不能代表总体分布时，BN 的归一化效果会显著退化[new-8]。若允许 BN 层在 Snipe 中更新运行统计量，将导致 Scout 阶段（使用全局统计量）和 Snipe 阶段（使用漂移后的统计量）之间出现特征表示不一致，进而引发训练不稳定甚至坍塌。

### 3.3.4 非对称设计的理论动机

Scout-Snipe 的核心设计动机是对经典 MIL 训练中一个脆弱环节的直接修补：在弱监督条件下，如果一边用尚不可靠的模型判断来选择实例，一边立刻让这些实例参与反向传播，就容易将早期的选择偏差迅速写入模型参数，形成自我强化的错误循环。

因此，本文采用"先探索、后学习"的非对称流程：搜索过程不携带梯度（Scout），学习过程只针对搜索结果中的高得分子集进行（Snipe）。两个阶段的解耦使得实例选择策略可以独立于梯度更新过程演化——即使 Scout 阶段选出了部分噪声实例，损失函数中的质量门控机制（见 3.5 节）也能在 Snipe 阶段对其进行分级处理，避免噪声信号对模型参数的直接污染。

## 3.4 模型架构

### 3.4.1 骨干网络选择与卷积架构论证

本文主线架构使用标准 CNN 骨干网络，并在实验中对比 EfficientNetV2-S（22.15M 参数）与 EfficientNet-B0（5.29M 参数）两种主干[17][18]，前者作为最佳性能配置，后者作为轻量部署候选。

**深度可分离卷积与标准卷积的特征表征差异**

选择骨干网络时，卷积算子的类型是一个需要明确论证的设计决策。标准卷积（Standard Convolution）的 $k \times k \times C_{in}$ 核在每次卷积运算中同时连接空间邻域和通道维度，构建的是一个空间-通道耦合的联合特征空间。深度可分离卷积（Depthwise Separable Convolution, DSC）则将这一运算严格分解为两个独立步骤：逐深度卷积（Depthwise Convolution）仅在单通道内提取空间特征，逐点卷积（Pointwise Convolution）通过 $1 \times 1$ 核进行跨通道线性混合[17][18]。这种分解使参数量从 $K^2 \cdot M \cdot N$ 降至 $K^2 \cdot M + M \cdot N$，在 $3 \times 3$ 卷积中可实现约 8-9 倍的参数压缩。

然而，这种参数效率的提升伴随着一个结构性代价：**通道关联与空间上下文的交叉项在深度可分离卷积中被拆解**。标准卷积在单次运算中即可学到"当某通道在此位置响应强、另一通道响应弱时，这种组合代表特定纹理"的联合模式；深度可分离卷积则必须通过逐深度→逐点两步串联来近似这种联合建模，而两步之间的信息传递是线性的、缺乏空间感受野的。

对于本项目的水稻病害检测任务，这一差异尤为关键。病灶的判别通常依赖颜色与纹理的联合特征——例如褐斑病（Brown Spot）表现为褐色坏死边缘镶嵌在绿色健康叶片组织中，细菌性叶枯病（BLB）表现为沿叶脉方向的黄白色萎蔫条纹。这些"特定颜色分布严格耦合在特定空间几何形状内"的复合病理特征，恰好需要标准卷积保留的跨通道空间联合结构来精确捕捉。

**EfficientNet 系列的内部架构演进：从 MBConv 到 Fused-MBConv**

理解 EfficientNet-B0 与 EfficientNetV2-S 之间的性能差异，需要深入其内部卷积模块的演进。

**EfficientNet-B0（V1 系列）** 全程采用 MBConv（Mobile Inverted Bottleneck Convolution）模块，其核心运算即为深度可分离卷积。这意味着从网络第一层到最后一层，空间特征提取与通道特征混合始终被分离处理。

**EfficientNetV2-S** 在架构设计上做出了一个重要修正：在网络浅层（Stages 0-2）使用 **Fused-MBConv** 模块替代了标准 MBConv[18]。Fused-MBConv 的关键改动是将深度可分离卷积"重新融合"为标准卷积——即将逐深度卷积和逐点卷积合并为一个标准的 $3 \times 3$ 或 $5 \times 5$ 卷积运算。Tan 与 Le 提出这一改动的原始动机是训练速度：深度可分离卷积虽然在理论 FLOPs 上更少，但在 GPU/TPU 等硬件加速器上无法充分利用张量核心的并行计算能力，反而导致实际训练吞吐量下降[18]。

然而，从特征表征的角度分析，Fused-MBConv 在浅层回归标准卷积带来了一个同样重要的副效应：**网络浅层（负责提取低层纹理、边缘、颜色等基础视觉特征）重新获得了完整的空间-通道联合建模能力**。而深层（Stages 3+）继续使用 MBConv 以保持参数效率，这在高层语义特征已足够抽象的条件下是合理的折中。

**冻结策略下的预训练特征继承**

本文的训练策略将 Stages 0-2 永久冻结（使用 ImageNet 预训练权重），仅解冻 Stages 3+ 在病害数据上进行微调。这一设计决策使得骨干网络选择的影响集中体现在**预训练阶段所学低层特征的质量**上：

- **V2-S 冻结层**（Fused-MBConv）：在 ImageNet 预训练阶段即使用标准卷积学习低层特征，因此其冻结权重携带了更完整的跨通道空间耦合表征。
- **B0 冻结层**（MBConv）：在 ImageNet 预训练阶段使用深度可分离卷积学习低层特征，其冻结权重中的跨通道耦合信息先天弱于 V2-S。

可训练的深层（Stages 3+）在两个骨干上接收到不同质量的低层输入——V2-S 的深层从一开始就获得了更高质量的特征基础。这为消融实验中 V2-S bare（90.03%）稳定优于 B0 bare（89.07%，差距 +0.96pp）提供了理论解释。

**消融结果对架构选型的验证**

消融实验进一步支撑了上述分析：

**B0 + FPN 的增益远大于 V2-S + FPN（+1.10pp vs +0.17pp）。** FPN 的自顶向下多尺度特征融合对 B0 起到了显著的补偿作用——B0 浅层的深度可分离卷积提供的多尺度信息先天不足，FPN 恰好弥补了这个短板。而 V2-S 的 Fused-MBConv 浅层已经提供了足够好的低层特征，FPN 的边际增益递减。

**ViT 对 V2-S 的损害远大于 B0（−2.74pp vs −0.75pp）。** V2-S 的高维特征空间（1792 通道）经过 FPN 降维到 256 通道后再接 ViT，信息瓶颈更为严重；同时 V2-S 已有的高质量特征表征被 ViT 的全局自注意力机制破坏的代价也更高。

**为何不选择 ResNet**

一个合理的追问是：既然标准卷积在特征耦合上具有优势，为何不直接选择全程标准卷积的 ResNet 作为骨干？原因在于 EfficientNet 系列的复合缩放策略（Compound Scaling）在同等参数量下提供了更好的精度-效率平衡[17]。ResNet-50（25.6M 参数）与 EfficientNetV2-S（22.15M 参数）参数量相近，但 EfficientNet 通过同时平衡网络深度、宽度与输入分辨率，在分类基准上实现了更高的参数效率。

更重要的是，EfficientNetV2-S 通过 Fused-MBConv 在浅层已经回归了标准卷积的耦合优势，同时在深层通过 MBConv 保持了参数效率——这是一种兼顾两者优势的折中设计。加上本文冻结浅层的训练策略，V2-S 实际上继承了标准卷积在低层特征提取上的核心优势，同时避免了 ResNet 在参数效率上的劣势。

最终，**消融结果是最直接的论证**：V2-S bare 在修正损失权重后达到 91.25% 的整体准确率，是所有配置中的最高值。这一结果本身即为骨干选择提供了最强的实证支撑。

### 3.4.2 架构配置一：Bare Backbone（EfficientNetV2-S 主线生产架构）

主线生产架构采用极简的分类头设计，仅包含骨干网络、一个 $1 \times 1$ 卷积分类器（`Conv2d(1792, 9, 1)`）与全局最大池化（GMP）。该架构在训练路径和推理路径上使用不同的运算顺序，但因 Conv1×1 的逐点线性特性，两条路径在数学上严格等价。

**训练数据流**

训练阶段的目标是为每个 tile 输出一个 9 维分类向量，采用"先池化后分类"的顺序：

```
训练路径 (Bare V2-S)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Input Tile (B, 3, 384, 384)
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  EfficientNetV2-RW-S Backbone                           │
│    Stages 0-2: FROZEN (Fused-MBConv, ImageNet预训练)    │
│    Stages 3-6: TRAINABLE (MBConv, 病害微调)             │
│    conv_head: Conv2d(272→1792) + BN (通道扩展)          │
│                                                         │
│  输出: 特征图 F ∈ ℝ^(B, 1792, 12, 12)                  │
└─────────────────────────────────────────────────────────┘
    │
    ▼
GMP: Global Max Pool (1792, 12, 12) → (1792, 1, 1)
    │  每通道取空间最大值，压缩空间维度
    ▼
Conv1×1: Conv2d(1792, 9, 1)
    │  逐点线性映射至 9 类
    ▼
Flatten → Logits (B, 9)
    │
    ▼
Loss Function (TopK Anchored MIL Loss)
```

GMP 体现了 MIL 的"存在性"语义：某一类别只要在特征图中任一空间位置出现足够强的响应，就可以主导该 tile 的类别判断。

**推理数据流（空间热力图生成）**

推理阶段需要保留空间信息以生成逐像素的类别响应图，因此**跳过 GMP**，直接将 Conv1×1 应用于未经池化的空间特征图上：

```
推理路径 (Bare V2-S) — 单 tile 热力图生成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Input Tile (B, 3, 384, 384)
    │
    ▼
EfficientNetV2-RW-S Backbone (同训练)
    │
    ▼
特征图 F ∈ ℝ^(B, 1792, 12, 12)
    │
    ▼   ┌──────────────────────────┐
    │   │  ✗ 不经过 GMP            │
    │   │  直接在空间维度上分类     │
    │   └──────────────────────────┘
    ▼
Conv1×1: Conv2d(1792, 9, 1)  ← 与训练使用同一组权重
    │
    ▼
Tile 级空间热力图 H_tile ∈ ℝ^(B, 9, 12, 12)
    │  每个空间位置 (i,j) 有 9 个类别得分
    │
    ▼ ── 输出至推理流水线（3.6 节）──
    │
    Resize 至该 tile 在原图中的区域尺寸
    → 加权累加至全图坐标系 accumulator
    → 多尺度多 tile 融合后得到全图热力图
```

> ⚠️ **需插入图：图3.4b** — Bare Backbone（V2-S）架构图，需展示训练/推理两条数据流。

**两条路径的权重共享与语义关联**

训练路径的计算为 $\text{logit}_c = \sum_k W_{c,k} \cdot \max_{(i,j)} f_k(i,j)$（逐通道取空间最大值，再线性组合），推理路径的计算为 $H_c(i,j) = \sum_k W_{c,k} \cdot f_k(i,j)$（在每个空间位置上做线性组合）。需要注意的是，这两条路径虽然共享完全相同的 Conv1×1 权重矩阵 $W$，但**并非严格的数学等价关系**：训练路径中的逐通道全局最大池化（GMP）允许不同通道在不同空间位置取得各自的最大值，其结果为来自多个空间位置的混合信号；而推理路径中每个位置 $(i,j)$ 的热力图值仅使用该位置的局部特征。两者之间不满足严格的可交换性（max 与线性组合不可交换）。

尽管如此，两条路径的非等价性并不构成设计缺陷，而恰恰体现了本文在训练端与推理端的分工设计意图。
训练路径中，GMP 的逐通道空间最大值选择起到了双重作用：其一，它实现了 tile 表征级别的最强响应主导——每个通道只将自身空间范围内最显著的激活贡献给分类决策，天然过滤了该通道内弱响应位置的噪声；其二，Conv1×1 在这些经过 GMP 筛选的最强响应上学习"什么通道组合模式对应什么病害类别"，使权重矩阵编码了类别判别性最强的通道关联结构。这一机制与损失函数层面的 Top-1 tile 杠杆原则（见 3.5 节）在不同粒度上共同服务于同一目标：让最具判别力的信号主导参数更新。
推理路径中，绕过 GMP 将 Conv1×1 逐位置施加于未经池化的空间特征图上，其目的不再是分类决策，而是生成空间响应图——标记出原图中每个局部区域对各病害类别的响应强度，为下游的热力图融合与候选检测框提取提供空间线索。训练好的权重矩阵在此处被复用为"类别响应探针"：它在训练阶段学会了识别判别性通道组合模式，推理阶段将同一组模式逐位置检测，输出的热力图即为这些模式在空间维度上的分布。
因此，两条路径的关系应理解为功能互补而非数学等价：训练路径负责"学会判别"，推理路径负责"画出可疑区域"，两者共享同一组 Conv1×1 权重，各司其职。这种"训练端优化全局判别 → 推理端复用权重生成空间响应"的设计范式在弱监督定位中被广泛采用（如 CAM 系列方法[12][13]），其空间响应的有效性最终由可视化结果和下游定位质量来验证（见第四章 4.6 节）。


### 3.4.3 架构配置二：FPN + HeatmapHead（EfficientNetV2-S 部分混合架构）

部分混合架构在 Bare Backbone 基础上引入特征金字塔网络（FPN）进行多尺度特征融合，并使用 HeatmapHead 替代原始的 GMP+Conv1×1 分类头。相比 Bare Backbone，该架构在训练和推理阶段采用统一的"先分类后池化"数据流，空间热力图作为显式中间产物始终可用。

**训练数据流**

```
训练路径 (V2-S + FPN + HeatmapHead, 无 ViT)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Input Tile (B, 3, 384, 384)
```

**ViT 放置位置的设计考量**

ViT 被放置在 FPN 降维后的 256 通道特征上（而非直接放在骨干网络的高维特征之后），这带来约 41× 的参数缩减（从 38.6M 降至 0.94M）。若直接在 1280 维（B0）或 1792 维（V2-S）特征上运行 ViT，MLP 隐藏层维度将达到 5120-7168，仅 ViT 模块即需约 38.6M 参数，在 8GB VRAM 的硬件上无法完成训练。ViT 采用残差连接设计，确保在注意力机制未能提供有效增益时，信息可以直接通过跳跃连接传递，不会损害 FPN 已有的特征表示。

ViT 使用可学习的 2D 位置编码（24×24 = 576 个 token），当推理时输入分辨率发生变化时，可通过双线性插值自适应调整位置编码的空间维度。

### 3.5.2 Warmup 阶段损失设计（Epoch 1-15）

$$\mathcal{L} = \mathcal{L}_{neg} + w(t) \cdot \lambda_1 \cdot \text{CE}(\text{Top-1}) + \lambda_2 \cdot \text{CE}(\text{Top-2\sim K}) + \lambda_r \cdot \mathcal{L}_{inter}$$

采样配置（等贡献原则）：以 8 bags × K=3 组织批次，每批包含 8 个 negative tiles、8 个 top-1 tiles 和 16 个 top-2~K tiles。各组分的损失权重经校准使三组样本对总损失的贡献近似相等（每组约 8.0）：

- **neg 组**：8 tiles × CE weight 1.0 = 8.0
- **top-1 组**：8 tiles × $\lambda_1$ = 8.0（$\lambda_1 = 1.0$）
- **top-2~K 组**：16 tiles × $\lambda_2$ = 8.0（$\lambda_2 = 0.5$）

在此等贡献基础上，动态权重 $w(t)$ 和 ranking loss 提供额外的 top-1 锚定增强，确保 top-1 tiles 在质量门控中的主导地位（杠杆原则）。

### 3.5.3 Stable 阶段的三级质量门控（Epoch 16-30）

进入 Stable 阶段后，损失函数对每个 bag 的 Top-1 tile 按预测质量进行三级分层，并对 Top-2~K tiles 施加差异化监督：

> ⚠️ **需插入表：表3.4** — 三级质量门控策略表

| Tier | 条件 | Top-1 处理 | Top-2~K 处理 |
|---|---|---|---|
| Tier 1 (Qualified) | 预测正确 AND conf > 0.45 | CE ×1.0 (dynamic) | Per-Bag KL (κ=0.5) |
| Tier 2 (Marginal) | 预测正确 AND conf ≤ 0.45 | CE ×1.0 (full weight) | Weak CE ×0.25 |
| Tier 3 (Wrong) | 预测错误 | Noise Drop / Correction Ranking | Silence (Loss=0) |

其中，Tier 1 的置信度门控阈值 0.45 远高于 9 类随机基线（≈0.11），表示模型对该预测具有足够的确信度，可作为 teacher 分布指导 Top-2~K tiles 的学习。

### 3.5.4 Per-Bag KL 蒸馏

Per-Bag KL 蒸馏的设计出发点是：若直接采用 batch-mean KL 在整个批次上做软标签蒸馏，不同 bag、不同类别的预测分布会相互混合，导致热力图出现"到处都像病害"的弥散问题。

本文引入双门控机制来保证蒸馏质量：
- **Gate A（bag-level）**：该 bag 的 Top-1 是否属于 Tier 1（teacher 分布可信）
- **Gate B（tile-level）**：当前 Top-2~K tile 的 argmax 是否与 bag 标签一致（student 方向正确）

只有当两个 gate 同时通过时，才使用该 bag 内 Top-1 的软分布作为 teacher 进行 KL 蒸馏；否则退回弱 CE 监督。

**Stable 阶段的 Tier 分配概率估算**

基于最佳配置（A3-corrected, V2-S, val_acc=91.25%）的实测数据，可以粗略估算 Stable 阶段各 Tier 的典型触发比例：Tier1 约占 90%（正确且高置信），Tier2 不足 1%（正确但低置信），Tier3 约占 9%（预测错误）。这意味着 KL 蒸馏在 Stable 阶段对绝大多数 bags 生效，是主要的监督信号来源；而 Tier3 的 Noise Drop / Correction Ranking 分支仅处理少量困难样本。

在 Scout-Snipe 非对称训练中，由于两个阶段的模型状态切换（eval → train）和批次构成的非自然性，存在若干特有的训练不稳定风险。本文设计了一套反坍塌机制加以应对：

> ⚠️ **需插入表：表3.5** — Anti-Collapse 机制汇总表

| 问题 | 根因 | 修复策略 |
|---|---|---|
| BN 统计量漂移 | Scout(eval) vs Snipe(train) 的 BN 状态不一致[new-7][new-8] | Snipe 中冻结 BN 层为 eval() 模式 |

## 3.6 推理流水线

### 3.6.1 自适应多尺度切片

推理阶段根据原图尺寸自适应选择 tile 尺寸集合。对每个尺度，采用步长为 tile_size × 0.5 的滑窗切片方式，保证相邻 tiles 之间有 50% 的空间重叠，从而实现对原图的充分覆盖。

> ⚠️ **需插入图：图3.7** — 推理流水线完整流程图（从原图输入到检测框输出）

### 3.6.2 尺度加权融合与热力图生成

对于每个 tile，模型输出 tile 级空间热力图（Bare Backbone 为 $9 \times 12 \times 12$，HeatmapHead 架构为 $9 \times 24 \times 24$）。该热力图首先通过双线性插值 resize 至该 tile 在原图中所覆盖区域的像素尺寸，然后加权累加至与原图同尺寸的全局 accumulator（$9 \times H_{\text{orig}} \times W_{\text{orig}}$）。所有尺度的所有 tile 处理完毕后，accumulator 除以对应的权重计数图得到最终全图热力图。

融合时使用三种权重/修正机制：
- **scale_weight = ts / max_ts**：较大尺度 tile 因覆盖更广的语境而获得更高权重
- **bg_aware_weight = max(0, 1 − 2×bg_prob)**：背景概率高的 tile 被降权，抑制噪声响应
- **feather_window（Hann window）**：在 tile 边界施加余弦衰减窗函数，减轻拼接边界伪影

融合后对每个 disease channel 执行 TopK 空间归一化（k=3），抑制弱响应噪声。

### 3.6.3 检测框提取

后处理步骤依次为：（1）percentile thresholding + Gaussian blur 平滑；（2）形态学开闭运算去除小噪点；（3）连通域分析（CCA）提取候选区域及其边界框；（4）Greedy NMS（IoU > 0.3）去除重叠检测；（5）Per-region class refinement（区域内类别重投票）；（6）BG-region filter（过滤背景主导区域）；（7）按 area × confidence 排序输出 top-k 检测框。

## 3.7 评估指标体系

### 3.7.1 混淆矩阵与基础指标定义

所有高级指标均派生自基础的混淆矩阵四元素：TP、TN、FP、FN。

> ⚠️ **需插入表：表3.6** — 评估指标数学定义汇总表

| 指标 | 公式 | 含义 |
|---|---|---|
| Accuracy | $\frac{TP+TN}{TP+TN+FP+FN}$ | 全局分类正确率 |
| Precision | $\frac{TP}{TP+FP}$ | 预测纯度（阳性预测中真阳性的比例） |
| Recall | $\frac{TP}{TP+FN}$ | 疾病敏感度（实际阳性中被正确检出的比例） |
| F1 | $\frac{2 \times P \times R}{P+R}$ | 精确率与召回率的调和平均 |

### 3.7.2 Bag 级指标

- **Overall Accuracy (filtered)**：基于 Hit Accuracy 过滤后的测试集全量推理整体准确率
- **Disease Macro-F1 (C1-C8)**：8 类病害的宏平均 F1
- **Avg Top-1 Confidence**：每个 bag 中最高响应 tile 预测的平均置信度
- **Hit Accuracy (bag-level)**：top-1 tile 正确预测为 bag 标签类别的比例

**Bag 级指标的正式定义**

**Overall Accuracy (filtered)**：在全量验证集上，对每张图像执行多尺度推理后，仅计算"Hit Accuracy 命中"的图像子集上的 tile 级分类准确率。其中"Hit"定义为该图像的 Top-1 tile 预测类别与图像级标签一致。

$$\text{Overall Acc (filtered)} = \frac{\sum_{i \in \text{HitBags}} \text{CorrectTiles}_i}{\sum_{i \in \text{HitBags}} \text{RetainedTiles}_i}$$

**Disease Macro-F1**：8 个病害类别（C1-C8）的 F1 分数的算术平均值，不包含背景类别。

$$\text{Disease Macro-F1} = \frac{1}{8} \sum_{c=1}^{8} F1_c$$

**Hit Accuracy (bag-level)**：在验证集 bags 中，Top-1 tile 的预测类别与 bag 标签一致的比例。

$$\text{Hit Acc} = \frac{|\{b : \text{argmax}(\text{Top1Score}_b) = y_b\}|}{|B_{\text{val}}|}$$

**Avg Top-1 Confidence**：所有验证集 bags 中，Top-1 tile 在目标类别上的 softmax 概率的平均值。

$$\text{Avg Top1 Conf} = \frac{1}{|B_{\text{val}}|} \sum_b P(\hat{y}_b = y_b \mid \text{Top1}_b)$$

### 3.7.3 Tile 级指标（过滤后）

- **Tiles Retained / Filter Rate**：模型最终保留的有效局部样本比例
- **Top-K Lift**：Top-K 平均置信度减去随机 tile 平均置信度
- **FG Ratio**：正样本 tiles 中预测为病害（非背景）的比例

**Tile 级指标的正式定义**

**Top-K Lift**：衡量 Scout 阶段 Top-K 选择策略相对于随机选择的增益。

$$\text{TopK Lift} = \text{mean}(\text{conf}(\text{TopK tiles})) - \text{mean}(\text{conf}(\text{Random tiles}))$$

**Tiles Retained / Filter Rate**：在全量推理中，预测为病害且置信度超过阈值的 tiles 占全部正样本 tiles 的比例。


### 3.7.4 负样本指标

- **Background Recognition (Neg Recall)**：负样本正确预测为 Class 0 的比例
- **Neg Disease Hallucination**：负样本上病害类最大 softmax 概率的均值

**负样本指标的正式定义**

**Background Recognition (Neg Recall)**：负样本 tiles 中 argmax 预测为 Class 0（背景）的比例。这是一个离散的二值指标，衡量模型是否将负样本正确归类为背景。

$$\text{Neg Recall} = \frac{\text{负样本中预测为 Class 0 的数量}}{\text{负样本总数}}$$

**Neg Disease Hallucination**：在所有负样本 tiles 上，计算每个 tile 的病害类最大 softmax 概率 $\max(P(\text{Class 1}), \ldots, P(\text{Class 8}))$，取所有负样本的均值。这是一个连续概率指标，衡量模型在健康背景上"幻觉出病害响应"的平均强度。

$$\text{Neg Hallucination} = \frac{1}{|N|} \sum_{i \in N} \max_{c=1}^{8} P(c \mid x_i)$$

注意 Neg Recall 与 Neg Disease Hallucination **不是互补关系**：一个 tile 即使被正确预测为 Class 0（argmax=0，贡献 Neg Recall），其病害类概率仍可能非零（例如 max disease prob=0.15，贡献 Hallucination）。Neg Recall 衡量的是"是否做出了正确的离散分类决策"，Hallucination 衡量的是"模型对病害类赋予了多少概率质量"。两个指标从不同角度监测模型是否出现"到处都看见病害"的过拟合倾向。

在 Warmup 阶段 P0 准入检查中，要求 Neg Recall > 0.70 且 Hallucination < 0.30，确保模型在进入 Stable 相位前已建立基本的背景抑制能力。在最佳配置中，Neg Recall 达到 99.15%，Hallucination 仅 0.96%，表明模型建立了极为清晰的病害与背景分类边界。

### 4.3.1 架构消融实验

#### 4.3.1.1 关键发现

**发现一：训练策略主导性能上限。** 最核心的发现是，纯骨干配置（Arch-1、Arch-2）在整体准确率上与混合架构配置（Arch-3、Arch-4）持平甚至更优。Arch-1（V2-S 纯骨干）达到 90.03%，而 Arch-3（V2-S 完整混合架构）仅 87.46%，简单架构反而高出 2.57 个百分点。这有力地表明，Scout-Snipe 训练框架本身才是性能的主要驱动因素，而非架构复杂度。

**发现二：跨骨干一致性。** V2-S 与 B0 在所有匹配配置下的性能差距非常小：Arch-1 vs Arch-2 仅差 0.96pp，Arch-5 vs Arch-6 仅差 0.03pp。这证实训练策略在不同骨干容量水平间具有良好的泛化性，也为仅在 B0 上进行损失和池化消融提供了方法论上的合理性。

**发现三：FPN 对两种骨干均提供多尺度感知补偿，且对 B0 的增益更为显著。** 将 FPN 的贡献单独隔离（纯骨干 → FPN only）：V2-S 从 90.03% → 90.20%（+0.17pp），B0 从 89.07% → 90.17%（+1.10pp）。一个与此一致的假说是：B0 的浅层全部采用 MBConv（深度可分离卷积），其多尺度空间特征提取能力相对受限，FPN 的自顶向下多尺度特征融合对其起到了更显著的补偿作用；而 V2-S 的 Fused-MBConv 浅层可能已经提供了较充分的多尺度低层特征，使 FPN 的边际增益递减。但当前实验设计并未单独控制浅层卷积类型这一变量，因此这一解释应被视为合理的方向性假说而非因果结论。

> 发现二和发现三的结果与以下推测一致：两个骨干的浅层卷积架构差异（B0 全程 MBConv vs V2-S 浅层 Fused-MBConv）可能导致了 FPN 增益的非对称性。但需注意，两个骨干在参数量、网络深度等维度上也存在差异，上述解释为可能的贡献因素之一。

**发现四：ViT 在两种骨干上均为负优化，且对 V2-S 的损害远大于 B0。** 将 ViT 的边际效应隔离（FPN only → FPN+ViT+Head）：V2-S 从 90.20% → 87.46%（−2.74pp），B0 从 90.17% → 89.42%（−0.75pp）。ViT 的全局自注意力机制不仅没有带来预期的全局上下文建模增益，反而降低了已有特征表征的质量。一种可能的解释是，ViT 缺乏卷积网络固有的局部性与平移不变性等归纳偏置[20]，在本项目的训练数据规模下不足以学习到这些先验，导致 ViT 模块引入了额外噪声。但该解释仍需进一步实验验证。

**发现五：组件的整体效果在两种骨干上呈现方向性差异。** 纯骨干与完整混合架构直接对比：V2-S 从 90.03% → 87.46%（−2.57pp，净负），B0 从 89.07% → 89.42%（+0.35pp，净正）。在 B0 上，FPN 的 +1.10pp 增益足以抵消 ViT 的 −0.75pp 损害；在 V2-S 上，FPN 仅提供 +0.17pp 的边际增益，远不足以补偿 ViT 的 −2.74pp 破坏。这揭示了一个重要原则：**组件的有效性依赖于骨干网络的容量语境**，同一组件在不同骨干上可能产生截然相反的净效果。

### 4.3.2 损失函数消融实验

损失函数消融旨在验证三级质量感知损失机制相比简单交叉熵是否提供有意义的改进。受控对比实验（Loss-1 至 Loss-3）在 B0+FPN（无 ViT）配置上进行，确保架构为唯一控制变量。此外，Loss-4 在 V2-S 纯骨干上验证权重修正在最佳架构上的迁移效果，其对比对象为 Arch-1（同架构、旧权重），不与 Loss-1~3 直接比较。Loss-5 为旧权重参考基线。

**权重修正说明（2026-03-09）**：
(a) Tier-2 的 Top-1 tile 缺失 dynamic_weight 乘数，导致 Tier2 和 Tier1 之间的监督强度层级倒置。
(b) top2k_soft_weight 被重复应用（0.3×0.1 = 0.03），使 Top-2~K 的损失贡献被压至极小值，违反了等贡献原则。
(c) 全部权重按等贡献原则（neg ≈ top1 ≈ top2k ≈ 8.0）重新校准。

> ⚠️ **需插入表：表4.3** — 损失函数消融实验结果

| 实验ID | 配置 | 损失模式 | Overall Acc (%) | Disease Macro-F1 (%) | Neg Recall (%) | Hit Acc (%) |
|---|---|---|---|---|---|---|
| Loss-1 | B0+FPN | Tiered（修正, λ₁=1.0） | 83.22 | 74.66 | 97.21 | 92.54 |
| Loss-2 | B0+FPN | Simple CE | 88.34 | 81.28 | 96.23 | 99.58 |
| Loss-3 | B0+FPN | Tiered（λ₁=2.0, dw 0.7→1.0） | 88.58 | 81.30 | 97.20 | 99.36 |
| Loss-4 | V2-S bare | Tiered（修正, λ₁=2.0） | 91.25 | 85.69 | 99.15 | 100.00 |
| Loss-5 | B0+Full | Simple CE（旧权重） | 89.19 | 82.51 | 97.61 | 99.68 |

#### 4.3.2.1 关键发现

**发现一：λ₁=1.0 时 Top-1 杠杆缺失导致类别坍塌。** Loss-1（tiered，λ₁=1.0）出现显著性能退化：整体准确率降至 83.22%，Hit Accuracy 从接近 100% 坍塌至 92.54%。混淆矩阵显示 Brown Spot 有约 64% 的 tiles 被误分至 Leaf Blast（BS F1 从 ~88% 骤降至 44.30%）。分析表明，当 Top-1 锚定权重与其他 tile 权重相等时，三级质量门控的前提被破坏——Tier1 的"高质量 teacher"与 Tier2/Tier3 的"低质量 student"在损失贡献上没有差异，模型失去了维持清晰类别边界所需的梯度杠杆。这直接验证了 **Top-1 杠杆原则** 的必要性。

> ⚠️ **需插入图：图4.3** — Loss-1 (λ₁=1.0) 的混淆矩阵，重点标注 C2→C4 混淆

**发现二：增强 Top-1 锚定权重可恢复性能。** Loss-3（λ₁=2.0，渐进式 dynamic_weight 0.7→1.0）达到 88.58% 准确率和 99.36% Hit Accuracy，从 Loss-1 的坍塌中完全恢复。这证实 Top-1 tile 必须获得至少 2 倍于其他样本的有效监督权重，才能在质量门控损失的层级结构中发挥可靠的锚定作用。

**发现三：Simple CE 是出人意料的强基线。** Loss-2（simple CE，对所有 Top-K tiles 施加统一 CE 损失，不做质量分层）达到 88.34% 准确率，与 Loss-3（88.58%）非常接近（差距仅 0.24pp）。这表明在 Scout-Snipe 框架内，Top-K 选择本身已经完成了主要的质量过滤工作，对质量过滤后的 tile 集合施加简单交叉熵已能捕获大部分收益；三级分层结构的主要附加价值来自 Top-1 杠杆机制对类别边界的增强，而非 tier 分层本身。

**发现四：权重修正显著提升 V2-S 纯骨干性能。** Loss-4（V2-S bare，修正权重 λ₁=2.0）达到所有实验中最高的整体准确率 **91.25%**，相较于 Arch-1（同架构、旧权重）的 90.03% 提升了 1.22pp。Disease Macro-F1 从 84.34% 提升至 85.69%（+1.35pp）。这一结果表明，正确的损失权重校准所产生的性能增益远超任何单一架构变化的效果。

**发现五：等贡献原则与杠杆原则缺一不可。** 等贡献原则确保 neg、top-1、top-2~K 三组样本的基础损失贡献量级对齐（修正了旧配置中 top-2~K 贡献仅为 0.03 的缺陷）；杠杆原则（λ₁ ≥ 2.0，通过 dynamic_weight 实现）确保 Top-1 在等贡献基础上进一步获得主导地位。两者必须同时满足——仅有等贡献但无杠杆（Loss-1）导致类别坍塌，有杠杆但无等贡献（旧配置）则使 top-2~K 的监督信号被压至可忽略。

逐类别数据清晰地展示了 Top-1 杠杆缺失的灾难性后果：Loss-1 中 Brown Spot 的 F1 从其他实验的 85-91% 骤降至 44.30%，伴随 64.3% 的 C2→C4 混淆率。这意味着在 λ₁=1.0 条件下，模型在 Brown Spot 和 Leaf Blast 之间丧失了类别区分能力。值得注意的是，Loss-1 在 BLB 上反而取得了最高 F1（76.05%），这可能因为 BLB 的视觉特征更为独特（沿叶脉的长条状枯黄），在 Top-1 杠杆缺失时受到的跨类别干扰较小。

### 4.3.3 池化消融实验

池化消融在 B0+FPN+ViT+HeatmapHead 架构上比较 HeatmapHead 内两种 pool_mode：GMP 与 TopK-Mean（k=3）。

> ⚠️ **需插入表：表4.5** — 池化消融实验结果

| 实验ID | 池化模式 | Overall Acc (%) | Disease Macro-F1 (%) | Neg Recall (%) | Neg Halluc. (%) | Hit Acc (%) |
|---|---|---|---|---|---|---|
| Pool-1 | GMP | 85.55 | 76.10 | 96.74 | 3.51 | 99.15 |
| Pool-2 (=Arch-4) | TopK-Mean (k=3) | 89.42 | 82.43 | 97.89 | 2.40 | 99.68 |

TopK-Mean 在整体准确率上超出 GMP 3.87pp，在 Disease Macro-F1 上超出 6.33pp。GMP 的"赢者通吃"梯度分配机制将学习信号集中在单个空间最大值上，在混合架构中由于 HeatmapHead 引入了额外的可学习参数（Conv1×1 空间分类器），这种极度集中的梯度流导致训练过程欠稳定。TopK-Mean（k=3）将梯度分布到 top-3 个空间位置，提供更平滑的优化地形，使 HeatmapHead 的参数更新更加稳健。

需要特别指出的是，此结果特指混合架构内 HeatmapHead 的池化模式选择。纯骨干实验使用的是骨干网络原生的 GMP 路径（无 HeatmapHead），其强劲性能（V2-S: 90.03%，B0: 89.07%）表明 GMP 本身并非内在劣势——而是 HeatmapHead 引入的额外可学习参数更受益于 TopK-Mean 提供的分散化梯度分布。

### 4.3.4 消融实验总结

消融实验得出以下核心结论：

第一，**Scout-Snipe 训练框架是性能的主导贡献因素**。纯骨干配置始终匹配甚至超越混合架构，表明非对称 MIL 训练协议——而非模型复杂度——驱动了系统的有效性。这一发现具有方法论意义：在弱监督场景中，训练过程的组织方式（如何选实例、如何分配监督强度）可能比增加模型参数更为关键。

第二，**FPN 在两种骨干上均提供正向的多尺度感知补偿**，对较小的 B0 骨干增益更为显著（+1.10pp vs +0.17pp）。然而 ViT 残差块在两种骨干上均产生负面效果，在 V2-S 上损害尤为严重（−2.74pp）。组件有效性取决于骨干网络的容量语境。

第三，**三级质量感知损失的有效性依赖于 Top-1 杠杆原则**（λ₁ ≥ 2.0）。缺乏充分的 Top-1 锚定会导致质量层级坍塌和灾难性的类别混淆。在正确的权重校准下，修正后的损失在 V2-S 纯骨干上达到了所有实验的最佳结果（91.25%）。

第四，**在混合架构内，TopK-Mean 池化显著优于 GMP**（+3.87pp），证实 HeatmapHead 的额外可学习组件受益于更平滑的梯度分布。但此结论不适用于纯骨干架构中的原生 GMP 路径。

## 4.5 负面结果分析

### 4.5.1 Feature Critic 的失败与启示

Feature Critic 模块旨在利用预计算的背景原型（通过 K-Means 聚类负样本池的预训练特征构建）来降低 Scout 阶段中背景类 tiles 的病害 logits，从而提高 Top-K 选择的纯度。然而，实验数据揭示了其根本性缺陷：

在 ImageNet 预训练的 EfficientNetV2-RW-S 特征空间中，正样本（病害 tiles）与负样本（背景 tiles）对背景原型的余弦相似度几乎完全重叠——正样本 mean similarity = 0.9345，负样本 mean similarity = 0.9336，差距仅为 −0.0009。推荐阈值 0.82 在压制 95% 负样本的同时，也会压制 93.6% 的正样本。这意味着 ImageNet 预训练特征对"水稻叶片上的病斑"与"健康水稻叶片"之间的判别信息极为有限——两者在预训练特征空间中的流形高度重叠。

**启示**：Feature Critic 若要在此类任务中发挥作用，需要在经过病害数据微调后的 backbone 特征空间重新构建原型，或者使用与 CNN 特征正交的判别特征（如颜色直方图、纹理描述子等传统视觉特征）。

### 4.5.2 Negative Rejection 导致的 Silence Spiral

Negative Rejection (NR) 损失对 Top-2~K tiles 施加"不要是病害"（$\mathcal{L}_{NR} = -\log(1 - \max_{c>0} p_c)$）的梯度信号。这一梯度方向与 Top-1 CE 的"成为正确病害类别"方向存在根本冲突。当两者在同一 batch 的同一 bag 中共存时，NR 梯度逐步压制正样本的病害响应，形成一个自我强化的负反馈循环：

1. Top-2~K 在 NR 梯度下预测逐渐偏向背景
2. 这些 tiles 在下一 epoch 的 Scout 中得分更低
3. 更少的高质量 positive tiles 被选入 Top-K
4. 模型的正样本检测能力持续衰减（Silence Spiral）

此问题在 Tier2 中尤为严重：Tier2 的 Top-1 置信度已经较低（预测正确但 conf ≤ 0.45），NR 的反向梯度进一步加速了信号衰减。最终解决方案是完全弃用 NR，改用弱交叉熵（weak CE，权重 0.25），保持所有 tiles 的梯度方向一致性——都朝向"成为正确类别"而非"不要是病害"。

### 4.5.3 Spatial NMS 的无效性

早期实验尝试在 Scout 阶段使用空间 NMS（基于 IoU 的非极大值抑制）去除重叠的 tiles，以增加 Top-K 选择的空间多样性。然而，由于离线切片的步长设置（stride = 0.75 × tile_size，即 25% 重叠），tiles 之间本身存在大量空间重叠。Spatial NMS 过于激进地过滤了高质量候选，反而降低了 Top-K 集合的总体质量。

最终该机制被 Scale Diversity 取代——通过对已选尺度的后续候选施加 0.5 的分数折扣来鼓励跨尺度多样性，而非硬性的空间去重。Scale Diversity 在保持 Top-K 质量的同时，有效促进了多尺度信息的覆盖。

## 4.6 可视化分析

本节通过若干训练/验证集之外的案例样本，使用最佳配置（A3-corrected）对模型的空间响应模式进行定性分析，展示其在外部图像上的可解释输出特征。这些结果应被视为案例级证据（case-level evidence），用于说明模型已学习到具有空间语义的类别响应模式，而非严格意义上的跨数据集泛化评估。

### 4.6.1 热力图可视化

> ⚠️ **需插入图：图4.4** — BLB 样本的逐尺度热力图（per_scale_heatmap_blb.png）。展示 768px / 1024px / 1536px 三个尺度下 BLB 通道的空间响应。

以一张训练集外的 BLB（细菌性叶枯病）样本为例，三个尺度下的 Per-Class 热力图可视化表明：BLB 类别通道在病叶区域沿叶脉方向产生了清晰且连贯的高值响应（JET colormap 中的红色-黄色区域），而背景区域（土壤、其他健康叶片）的响应保持在接近零的水平。这一结果有两方面意义：第一，模型在仅有图像级标签的训练条件下，确实学习到了具有空间语义的类别响应模式，能够将病灶响应定位到正确的物理区域；第二，不同尺度的热力图呈现互补特征——768px（小尺度）的响应更为密集且细粒度，捕捉了局部病斑纹理细节；1536px（大尺度）的响应更加集中在高置信区域，覆盖了更完整的病灶结构。

### 4.6.3 检测框可视化与聚合瓶颈

> ⚠️ **需插入图：图4.7** — 检测框可视化（1detect_box.png）。展示 NMS 前后的检测框对比。

检测框可视化展示了推理流水线从连通域分析（CCA）到最终输出的完整后处理过程。在本例中，CCA 阶段产生了 111 个原始候选框，经过 NMS 移除 106 个重叠框后保留 5 个最终检测。

最终检测框的结果揭示了当前推理流水线的核心瓶颈：尽管 tile 级分类模型在每个尺度上都以近乎 100% 的置信度正确识别了 BLB，但多尺度融合后的检测框中混入了其他类别的候选（如 Node Neck Blast 78.1%、Sheath Blight 93.4%、Rice Leaf Folder 94.4%）。其根本原因在于，当前的静态阈值化与 NMS 流程无法充分利用 tile 级分类的高确定性信息——不同类别的热力图在空间叠加后，弱响应通过 scale_weight 和 bg_aware_weight 的加权累积被放大，最终通过 CCA 阈值产生虚假检测框。

这一发现指向了明确的改进方向：引入具有记忆和状态感知能力的自适应推理模型（见第五章 5.4.1），使系统能够根据 tile 级分类的高置信判断动态调整热力图聚合策略，而非依赖固定阈值和静态后处理规则。

### 4.6.4 混淆矩阵分析

> ⚠️ **需插入图：图4.8** — 最佳配置（A3-corrected）的混淆矩阵热力图

混淆矩阵分析揭示了几个值得注意的混淆模式：

（1）**BLB 与 LB 的双向交叉混淆**：细菌性叶枯病（BLB）与叶瘟（Leaf Blast）之间存在双向交叉响应，两者在叶面延伸性病变的形态特征上具有视觉相似性，这与 4.6.1 节热力图可视化中观察到的交叉激活现象一致。

（2）**False Smut 的低召回率**：稻曲病（False Smut）的召回率相对较低（68.9%），其球状孢子簇在某些尺度下的视觉显著性不足，可能导致 Scout 阶段选择的 Top-K tiles 中混入非病灶区域。

（3）**背景类别的近乎完美识别**：背景类的 F1 达到 99.5%，负样本的平均最大病害概率（Hallucination）仅 0.96%，说明基于 confirmed-clean 负样本池的训练策略有效建立了稳固的病害/背景分类边界。

## 4.7 讨论

本章实验结果揭示了若干值得深入讨论的问题。

**训练策略 vs 架构复杂度。** 消融实验最一致的发现是：Scout-Snipe 非对称 MIL 训练协议——而非更复杂的模型架构——是系统性能的主要驱动因素。这一发现具有方法论意义：在弱监督场景中，如何组织训练过程（怎样选择实例、怎样分配监督强度）可能比增加模型参数更为关键。纯骨干配置在 6 组架构实验中始终保持竞争力甚至领先，表明 Scout-Snipe 框架对骨干选择具有较强的鲁棒性。

**骨干架构的隐性影响。** EfficientNetV2-S 相较于 B0 的 +0.96pp 准确率优势，不能仅归因于更大的参数量（22.15M vs 5.29M）。两个骨干在浅层卷积模块上存在本质差异——V2-S 使用 Fused-MBConv（回归标准卷积），B0 使用 MBConv（深度可分离卷积）——这一差异可能导致冻结的预训练浅层向深层传递了不同质量的低层特征基础。FPN 增益的非对称性（B0 +1.10pp vs V2-S +0.17pp）与此推测一致，但如 4.3.1 节所述，两个骨干在参数量、网络深度等维度上也存在差异，浅层卷积类型并非唯一混淆因素。

**损失设计的敏感性与权重校准的关键性。** 权重修正带来了 1.22pp 的准确率提升和 1.35pp 的 Macro-F1 提升——远超任何单一架构变化的影响。更具警示意义的是，错误的权重配置（如 top2k_soft_weight 的重复应用导致有效权重仅 0.03）不会导致训练失败（模型仍能收敛），但会以隐性方式严重损害分类质量。这说明在弱监督 MIL 框架中，损失函数的权重校准是一个高敏感性、低可观测性的关键参数——需要通过系统消融而非经验调参来验证其正确性。

**Tile 级分类 vs 检测框聚合的性能鸿沟。** 可视化分析中的案例证据提示：tile 级分类模型的实际判别能力可能远超 bag 级 F1 指标所反映的水平。以 BLB 为例，bag 级 F1 仅为 66.9%，但在所展示的案例中，tile 级分类在各尺度下均表现出接近 100% 的置信度。这一差距的可能根源在于多尺度热力图融合阶段：当不同类别的弱响应在空间维度叠加时，静态阈值化和 NMS 流程缺乏足够的上下文信息来抑制次要类别的噪声。这从反面提示了自适应感知-决策推理模型的潜在价值——一个能够利用 tile 级高置信度信息来动态指导热力图聚合的推理策略，有望缩小 tile 级分类能力与最终检测框质量之间的差距。

**资源效率。** 最佳配置使用的 EfficientNetV2-S 纯骨干仅有 22.15M 参数，训练 VRAM 消耗约 2.4GB，推理 VRAM 消耗约 1.0GB，远在单卡消费级 GPU（8GB VRAM）的预算之内。这表明在资源受限的硬件条件下实现弱监督病害检测是完全可行的。EfficientNet-B0（5.29M 参数）在准确率仅低约 2pp 的条件下提供了约 4.2× 的参数缩减，适合作为边缘设备部署场景的候选方案。

**局限性与展望。** 当前系统的主要性能瓶颈不在分类模型本身，而在推理流水线的多尺度聚合层。由于缺乏像素级真值标注，无法直接报告 mAP 或 IoU 等标准目标检测指标，评估仍依赖 bag 级和 tile 级的代理指标。未来工作的优先方向是引入自适应推理机制（见第五章 5.4.1），使系统能够动态利用 tile 级分类信号来优化热力图聚合与检测框提取，从而缩小 tile 级分类能力与最终系统输出之间的性能差距。

综合以上实验与可视化结果，本文认为：在仅使用图像级单标签的条件下，Scout-Snipe 框架已经实现了稳定的多病害分类（最佳配置 91.25% 整体准确率、85.69% Disease Macro-F1），并能够输出具有空间意义的热力图与候选检测框。当前系统的主要改进空间位于全图级热力图融合与检测框聚合阶段，而非 tile 级病害表征学习本身——这一判断与第一章提出的研究目标形成闭环，同时也为第五章的未来工作方向提供了明确的优先级依据。

# 第五章 结论与未来工作

## 5.1 结论

本文围绕“在仅使用图像级单标签的条件下，能否同时实现水稻病害分类与空间定位线索生成”这一核心问题，提出并实现了一个基于弱监督多示例学习（MIL）的水稻病害分析框架。与像素级分割任务不同，本文的输出形式为原图级空间热力图与候选检测框，其目标是在低标注成本条件下提供具有空间解释意义的病灶定位线索，而非恢复病灶的精确像素边界。

围绕这一目标，本文得到以下主要结论。

第一，Scout-Snipe 非对称 MIL 训练框架能够有效缓解弱监督场景中的标签噪声问题。通过将“搜索关键实例”与“利用关键实例更新参数”解耦为两个时序上分离的阶段，模型避免了训练早期由低质量实例引发的梯度污染。系统消融结果表明，真正主导性能提升的关键因素并非更复杂的混合架构，而是训练协议本身。在 EfficientNetV2-S 裸骨干配置下，系统取得了 91.25% 的整体准确率和 85.69% 的 Disease Macro-F1，为全部实验配置中的最优结果。

第二，三级质量感知损失与 Per-Bag KL 蒸馏机制能够提升弱监督训练的稳定性，并改善 bag 内监督信号的组织方式。特别是，Per-Bag KL 通过双门控机制将软标签传播严格限制在 bag 内部，从而避免跨 bag、跨类别的错误蒸馏造成热力图弥散。相关消融结果进一步表明，top-1 实例在弱监督训练中具有显著更高的监督杠杆作用；若对不同层级实例施加近似等权的监督，模型可能出现明显的类别坍塌现象。

第三，本文构建了从多尺度切片、热力图融合到候选检测框提取的完整弱监督推理流水线，使仅有图像级标签的模型能够输出具有空间意义的定位结果。实验与可视化分析显示，当前系统已经能够在图像级分类之外生成原图级热力图及检测框，但其主要瓶颈并不在 tile 级病害表征学习本身，而在多尺度热力图融合与后处理聚合阶段。以 BLB 为例，tile 级分类在多个尺度下均表现出极高置信度，而最终检测框质量仍受静态阈值化与 NMS 聚合规则的限制。这表明，当前系统已经基本具备“识别关键病害区域”的能力，但尚未完全解决“如何将这些局部高置信信号稳定组织为更高质量最终检测输出”的问题。

第四，本文验证了在单卡消费级 GPU 条件下实现弱监督水稻病害定位系统的可行性。最佳配置 EfficientNetV2-S 裸骨干的参数量为 22.15M，训练显存约 2.4GB，推理显存约 1.0GB；同时，EfficientNet-B0 在精度仅下降约 2 个百分点的条件下提供了约 4 倍的参数压缩，说明本文框架具有进一步面向部署场景压缩与迁移的潜力。

总体而言，本文的主要价值不在于提出一个能够替代全监督检测或像素级分割的完整终极方案，而在于证明：在图像级弱标签、复杂农田背景与单卡资源约束条件下，基于 MIL 的训练策略创新可以有效支撑水稻病害的多类别分类与空间响应生成；同时，实验结果也清晰揭示了当前系统的主要改进空间位于推理聚合层，而非基础分类能力本身。

## 5.2 研究贡献总结

结合全文的方法设计与实验验证，本文的研究贡献可概括为以下四个方面。

1. **提出面向水稻病害弱监督定位的 Scout-Snipe 非对称 MIL 框架。**  
   本文将实例搜索与参数学习显式解耦，针对农业场景中正袋噪声实例多、实例质量差异大的特点，提出了以训练协议为核心的弱监督建模方案。与标准端到端 MIL 不同，该框架强调先搜索、后学习的非对称时序组织，从训练机制层面降低伪正实例对优化过程的污染。

2. **提出三级质量感知监督与 Per-Bag KL 蒸馏机制。**  
   本文并未将 bag 内所有实例等价处理，而是依据实例质量对监督强度进行分层控制，并引入仅在 bag 内传播的软标签蒸馏策略，从而在不引入额外教师模型的条件下提高训练稳定性，避免热力图空间响应的跨类别弥散。

3. **实现完整的弱监督推理流水线。**  
   本文从多尺度切片、尺度加权融合、背景感知重权重到候选检测框提取，构建了完整的原图级推理流程，使图像级标签模型能够输出热力图与候选检测框，而不仅停留于 bag 级类别判定。该流水线为弱监督农业病害定位提供了可运行的系统实现路径。

4. **通过系统性消融实验与负面结果分析明确了方法边界。**  
   本文不仅验证了各设计组件的有效性，还对若干失败尝试进行了系统记录，包括 Feature Critic 的失效、Negative Rejection 导致的 Silence Spiral，以及 Spatial NMS 的有限效果。相关结果表明，当前系统的关键提升主要来自训练协议，而非混合架构复杂度本身；同时也为后续研究明确了优先优化方向。

## 5.3 研究局限性

尽管本文在弱监督水稻病害分类与空间定位方面取得了阶段性结果，但当前研究仍存在以下局限。

第一，本文缺乏像素级掩码或人工边界框等局部真值标注，因此无法直接报告 mAP、IoU 等标准目标检测或语义分割指标。当前对空间定位质量的评估仍主要依赖 bag 级与 tile 级间接指标，以及热力图与检测框可视化结果。这使得本文较难与全监督检测或分割方法进行严格的一一对应定量比较。

第二，本文当前采用的是单主导病害假设，即默认每张正样本图像只对应一种主要病害类别。该假设有助于简化建模与实验分析，但与真实农田环境并不完全一致。在实际场景中，同一叶片或同一图像中可能存在多种病害共存、病害与虫害共存，甚至病害与背景胁迫交织的复杂情况，这要求未来引入多标签 MIL 或更细粒度的区域级建模机制。

第三，当前系统的主要瓶颈已较清楚地暴露在推理聚合层。虽然 tile 级分类能力较强，且在训练集外样本上也能产生有意义的高置信局部响应，但形态学相似类别之间的热力图交叉激活、不同尺度弱响应的空间叠加，以及静态阈值化与 NMS 规则的局限，仍会在最终检测框中引入非主导类别候选。换言之，当前系统更接近于“已具备强局部识别能力，但全图级空间组织仍需优化”的状态。

第四，本文的定量评估主要建立在单一数据来源的训练/验证划分之上。尽管可视化结果显示模型在训练集外图像上仍能输出有意义的热力图和正确的 tile 级分类，但尚未建立跨数据集、跨设备、跨采集条件的系统泛化评估基准。因此，当前结论更适合被理解为“在现有数据条件下验证了方法可行性”，而不是对复杂真实农田环境下普适性能的最终证明。

## 5.4 未来工作建议

### 5.4.1 自适应感知-决策循环推理

当前系统采用的是全覆盖式多尺度滑窗推理。该策略虽然能够最大限度保证病灶区域不被遗漏，但其计算成本较高，且在热力图聚合阶段依赖固定阈值与静态后处理规则，难以充分利用 tile 级分类中已经存在的高置信局部信息。针对这一问题，未来工作的首要方向应是构建**自适应感知-决策循环推理模型**，推动系统从“静态网格扫描”向“主动动态采样”演进。

该方向的核心思路是：以当前全覆盖推理结果作为监督信号，训练一个轻量决策模块，使系统能够从低分辨率粗粒度观察开始，仅在高熵、高不确定性或高潜在病灶价值的区域执行更高分辨率的局部重扫。一个典型的决策循环可表述为：首先对整幅图像进行一次低分辨率粗扫描，得到初始 heatmap 与 disease entropy map；随后由决策模块读取当前空间统计量、类别置信度、历史一致性变化与预算信息，输出 zoom_in、zoom_out 或 stop 等动作；系统再对选定区域执行局部精细扫描并更新 heatmap，直至达到预算上限、熵收敛或高置信区域覆盖完成。该方向的目标并非简单减少计算量，而是在维持检测一致性的前提下，以更少的 tile 数量获得更高质量的最终定位结果。

从工程实现角度看，该自适应模块可以作为现有推理引擎之上的独立决策层逐步接入，先以 MLP + 显式历史缓存验证闭环逻辑，再逐步扩展至带有时序记忆的 GRU 模型，最终探索与特征流形深度耦合的轻量策略网络。这样的递进式路线有利于在不推翻现有系统的前提下逐步验证“感知—决策—再感知”的闭环推理机制。

### 5.4.2 多模态扩展

本文当前系统仅处理 RGB 图像输入，而病害在近红外（NIR）、多光谱或时间序列观测中可能呈现出更具判别性的光谱与动态变化模式。因此，未来可探索在弱监督框架下引入多模态输入融合，使空间定位不再仅依赖可见光纹理与颜色差异，而能进一步利用植被指数、叶绿素相关指标或生长阶段变化等信息。

在具体实现上，一条可行路径是先从“RGB + 派生指数特征”的浅层融合开始，考察多模态是否能在弱监督设定下提供更稳定的 bag 级分类与更清晰的空间响应；进一步再考虑特征层融合、跨模态注意力或时序建模，使系统逐步从静态单帧病害识别扩展到面向农田连续监测的早期预警分析框架。对于农业场景而言，多模态扩展的意义不仅在于提升单次识别精度，更在于增强模型对复杂环境变化和早期轻微病斑的感知能力。

### 5.4.3 轻量化部署

实验结果表明，本文框架在单卡 RTX 4060 Laptop GPU 上已具备较好的资源效率，其中 EfficientNet-B0 在仅带来有限精度下降的情况下显著降低了参数规模，说明系统已经具备进一步轻量化和部署优化的现实基础。未来工作可围绕模型压缩、推理加速与平台迁移三个层面展开。

首先，在模型层面可探索知识蒸馏、剪枝、量化与低秩分解等轻量化技术，将当前有效的弱监督训练策略迁移到更小型骨干网络上；其次，在系统层面可针对推理引擎进行 batch 组织优化、缓存复用与局部区域增量更新，以减少多尺度滑窗造成的重复计算；最后，在应用层面可进一步面向移动端、边缘端或无人机巡检场景开展适配研究，验证该框架在真实农田部署条件下的时延、功耗与稳定性表现。轻量化部署不应被理解为单纯的模型缩小，而应被视为将“弱监督可行性”真正转化为“场景可用性”的关键一步。
